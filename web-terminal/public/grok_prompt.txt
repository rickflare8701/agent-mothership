### 2. Service Architecture Refined
- STDispatch$Shavlik Protect: RUNNING (PID 4188, SYSTEM, auto-start). Image: STDispatch.exe
- STAgent$Shavlik Protect: STOPPED (PID 0, NetworkService). This is the main agent that does actual patching
- STDispatch listens on 5 interfaces (6 endpoints: ST.DispatchEvents, STDisp-EventSink, STDisp-FTQ, STN.Dispatch, STN.Core, STN.Core.Security)
- dataCache.dat: Format = 16-byte header (uint32_len + "Data" magic + uint32_field1 + uint32_padding) + UTF16-LE JSON event data
- store.dat: 0 bytes (agent unregistered, empty agentId in AgentEnvironment.config)
- Named pipes: NO ST/Dispatch pipes found (pure ncalrpc ALPC)
- WMI: NO ST/Shavlik/LANDESK providers found

### 3. RPC Endpoints — All Bind Without ACCESS_DENIED
Tested via Python ctypes and C# P/Invoke:
- All 6 endpoints: RpcBindingFromStringBinding succeeds (no 0x5 ACCESS_DENIED)

## Blockers & Gaps

1. ExE admin check bypass — Need to reverse engineer the admin check in STAgentCtl.exe (304KB native C++, uses OpenProcessToken but no CheckTokenMembership or IsUserAnAdmin imports). What's the simplest patch? Change a conditional jump, NOP the check, or patch the binary to always return admin?

2. RPC MIDL format strings — Can we extract the NDR format strings from STAgentFramework.dll or STDispatch.exe binary? The DLL is 2.1MB, EXE is ~1MB. The format strings define the marshaling for DispatchTask, DispatchCheckInAndUpdateAll, etc. If we can find them, we can make direct RPC calls bypassing COM.

3. COM activation without registry — Is there a way to activate the STDispatch COM objects without registry entries? Registration-free COM manifest? Activation context? The DLLs are present, interfaces defined, but no CLSID registered.

4. Original STAgent on unregistered state — Is STDispatch intentionally idle because store.dat is 0 bytes (agent unregistered)? If we register the agent (via register command or by writing a valid store.dat), would it start processing New\ and FTQ\ files?

5. Control code triggering — What do control codes 128 and 255 actually DO to STDispatch? Could one of them trigger file processing?
